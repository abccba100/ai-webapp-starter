# 🧩 AI 기반 웹앱 MVP 생성기

## MVP 목표 (한 문장)
비개발자가 “만들고 싶은 웹앱”을 자연어로 설명하면 → AI가 기능 명세 초안을 만들고 → 사용자가 이를 수정·확정한 뒤 → 디자인 시안을 선택하면 → **Next.js 기반 프로젝트 뼈대 코드가 자동 생성되고** → **미리보기 링크로 바로 확인할 수 있는 웹앱 MVP 생성기**.

---

## 🛠️ MVP에 포함되는 기능 (필수)

### 1. 프로젝트 생성
- **입력 항목**
  - 프로젝트 이름
  - 한 줄 설명 (자연어)
- 사용자는 개발 지식 없이 텍스트만 입력

---

### 2. 요구 입력 (자연어 기반)
- “만들고 싶은 프로그램”을 자유롭게 서술
  - 예:  
    > “스터디 모집을 관리하는 간단한 웹앱을 만들고 싶어요.  
    > 사용자는 글을 올리고, 관리자는 승인만 하면 돼요.”

---

### 3. AI 기능 분석 → 기능 명세 초안 생성
AI가 입력된 자연어를 분석하여 기능 명세 초안을 자동 생성

- **기능 리스트 구조**
  - 🔹 핵심 기능 (Core)
  - 🔸 부가 기능 (Optional)
  - 🔧 관리자 기능 (Admin)

- 각 기능은 간단한 설명 포함
  - 예:
    - [핵심] 게시글 작성: 사용자가 모집 글을 작성할 수 있음
    - [관리자] 게시글 승인: 관리자가 게시글 공개 여부를 결정

---

### 4. 명세 편집 및 확정
사용자는 생성된 기능 명세를 직접 수정 가능

- 기능 **추가 / 삭제 / 수정**
- 각 기능에 **우선순위 지정**
  - `P0` : 반드시 필요한 기능
  - `P1` : 있으면 좋은 기능

👉 이 단계에서 **최종 개발 기준 명세 확정**

---

### 5. 디자인 시안 선택 (프리셋 3종)
실제 디자인 작업 전, **UI 성격 선택**

- 카드 UI 형태로 선택
  1. **Minimal** – 단순, 여백 중심
  2. **Business** – 정보 중심, 안정적
  3. **Trendy** – 최신 UI, 강조 컬러

※ 실제 이미지를 생성하지 않고, **템플릿 기반 UI 스타일만 결정**

---

### 6. 코드 스캐폴딩 자동 생성
확정된 명세를 기반으로 프로젝트 뼈대 코드 자동 생성

- **기술 스택**
  - Frontend: Next.js (App Router)
  - Backend 대체: Next.js API Routes
  - DB: Supabase 연결 구조

- 포함 내용
  - 기본 폴더 구조
  - Supabase 연결 설정 파일
  - API route 예제
  - 환경변수 템플릿

---

### 7. 명세 기반 페이지 라우팅 자동 생성
기능 명세를 기준으로 페이지 자동 생성

- 예:
  - `/login`
  - `/posts`
  - `/admin`

※ 실제 기능 구현은 없고, **빈 페이지라도 자동 생성**

---

### 8. 미리보기 배포
- 생성된 프로젝트를 **Vercel로 자동 배포**
- 사용자에게 **미리보기 링크 제공**
- 별도 환경 설정 없이 즉시 확인 가능

---

## 🚫 MVP에서 제외되는 기능 (추후 확장)

다음 기능들은 MVP 범위를 벗어나므로 포함하지 않음

- ❌ AI가 다른 AI에게 기능별 개발 분배 후 완성까지 자동화
- ❌ 고급 디자인 자동 생성 (이미지 생성, Figma 수준)
- ❌ 결제 시스템, 권한 관리, 복잡한 워크플로우 자동 생성

---

## 🎯 MVP의 핵심 가치
- **비개발자도 “말로 시작해서 코드까지” 도달**
- 완성도가 아닌 **“시작 가능한 상태”를 빠르게 제공**
- 아이디어 → 구조 → 코드 → 배포까지의 진입 장벽 제거

---

## 📌 향후 확장 아이디어 (After MVP)
- 기능별 코드 자동 생성
- 권한/인증 템플릿
- AI 기반 UI 개선 제안
- 협업용 프로젝트 공유 기능

---

> 이 프로젝트의 목표는  
> **“개발을 대신해주는 AI”가 아니라  
> “개발을 시작하게 해주는 AI”입니다.**
